1. Арифметические операции
   - ADD: Сложение двух операндов
   - SUB: Вычитание одного операнда из другого
   - MUL: Умножение двух операндов
   - DIV: Деление одного операнда на другой.
   - INC: Увеличение значения операнда на единицу.
   - DEC: Уменьшение значения операнда на единицу.
   - NEG: Изменение знака операнда на противоположный.
   - ADC: Сложение с учетом переноса из предыдущего результата.
   - SBB: Вычитание с учетом заимствования из предыдущего результата.
   - IMUL: Умножение с расширением знака.
   - IDIV: Деление с расширением знака

2. Логические инструкции
   - AND: Логическое 'И' между двумя операндами. Пример: AND EAX, EBX - результат будет храниться в EAX.
   - OR: Логическое 'ИЛИ' между двумя операндами. Пример: OR EAX, EBX - результат будет храниться в EAX.
   - XOR: Логическое 'исключающее ИЛИ' между двумя операндами. Пример: XOR EAX, EBX - результат будет храниться в EAX.
   - NOT: Инверсия всех битов операнда. Пример: NOT EAX - инвертирует все биты в EAX.
   - TEST: Проверка установленных битов между двумя операндами без сохранения результата. Пример: TEST EAX, EBX - результат не сохраняется.
   - SHL/SAL: Сдвиг битов операнда влево. Пример: SHL EAX, 1 - сдвигает биты EAX на одну позицию влево.
   - SHR: Логический сдвиг битов операнда вправо. Пример: SHR EAX, 1 - сдвигает биты EAX на одну позицию вправо.
   - SAR: Арифметический сдвиг битов операнда вправо. Пример: SAR EAX, 1 - сдвигает биты EAX на одну позицию вправо, сохраняя зна

3. Инструкции пересылки данных
   - MOV: Копирует данные из одного места в другое. Пример: MOV EAX, EBX - копирует данные из EBX в EAX.
   - PUSH: Помещает данные из регистра или памяти на вершину стека. Пример: PUSH EAX - помещает значение EAX на стек.
   - POP: Извлекает данные с вершины стека в регистр или память. Пример: POP EAX - извлекает значение с вершины стека в EAX.
   - LEA: Загружает эффективный адрес указанной памяти в регистр. Пример: LEA EAX, [EBX+ECX*4] - загружает адрес выражения в EAX.
   - XCHG: Обменивает данные между двумя операндами. Пример: XCHG EAX, EBX - обменивает значения EAX и EBX.
   - IN: Считывает данные из порта ввода/вывода в регистр. Пример: IN AL, DX - считывает данные из порта с номером DX в AL.
   - OUT: Передает данные из регистра в порт ввода/вывода. Пример: OUT DX, AL - передает данные из AL в порт с номером DX
  
4. Инструкции управления потоком
   - JMP: Безусловный переход на указанный адрес. Пример: JMP label - переходит к метке label.
   - JE/JZ: Переход, если равно или ноль. Пример: JE label - переходит к метке label, если предыдущая операция дала результат равный нулю.
   - JNE/JNZ: Переход, если не равно или не ноль. Пример: JNE label - переходит к метке label, если предыдущая операция не дала результата равного нулю.
   - JA/JNBE: Переход, если больше или не меньше и не равно. Пример: JA label - переходит к метке label, если предыдущая операция дала результат больше нуля.
   - JB/JNAE: Переход, если меньше или не больше и не равно. Пример: JB label - переходит к метке label, если предыдущая операция дала результат меньше нуля.
   - JG/JNLE: Переход, если больше или не меньше и не равно с учетом знака. Пример: JG label - переходит к метке label, если предыдущая операция дала результат больше нуля с учетом знака.
   - JL/JNGE: Переход, если меньше или не больше и не равно с учетом знака. Пример: JL label - переходит к метке label, если предыдущая операция дала результат меньше нуля с учетом знака.
   - LOOP: Выполняет переход на метку, пока счетчик не достигнет нуля. Пример: LOOP label - уменьшает счетчик и переходит к метке label, если счетчик не равен нулю.
   - CALL: Вызывает подпрограмму по указанному адресу. Пример: CALL procedure - сохраняет адрес возврата и переходит к подпрограмме procedure.
   - RET: Возвращает управление из подпрограммы. Пример: RET - возвращает управление в точку вызова подпрограммы
  
5. Строковые интсрукции
   - MOVS/MOVSB/MOVSW/MOVSD/MOVSQ: Копируют данные из одного места в другое. MOVSB копирует байт, MOVSW - слово, MOVSD - двойное слово, MOVSQ - квадро-слово.
   - CMPS/CMPSB/CMPSW/CMPSD/CMPSQ: Сравнивают два элемента данных. CMPSB сравнивает байты, CMPSW - слова, CMPSD - двойные слова, CMPSQ - квадро-слова.
   - SCAS/SCASB/SCASW/SCASD/SCASQ: Сканируют строку в поисках байта, слова, двойного слова или квадро-слова.
   - LODS/LODSB/LODSW/LODSD/LODSQ: Загружают строку в регистр AL, AX, EAX или RAX.
   - STOS/STOSB/STOSW/STOSD/STOSQ: Записывают байт, слово, двойное слово или квадро-слово из регистра AL, AX, EAX или RAX в строку
  
6. Инструкции для работы с памятью
   - MOV: Перемещает данные между регистрами, памятью и непосредственными операндами. Пример: MOV EAX, [EBX] - копирует данные из памяти по адресу, на который указывает EBX, в регистр EAX.
   - PUSH: Записывает данные из регистра или памяти в стек. Пример: PUSH EAX - помещает содержимое регистра EAX на вершину стека.
   - POP: Извлекает данные из стека в регистр или память. Пример: POP EAX - извлекает значение с вершины стека и помещает его в регистр EAX.
   - LEA: Загружает адрес операнда в регистр. Пример: LEA EAX, [EBX+ECX*4] - помещает в EAX адрес, вычисленный по формуле [EBX+ECX*4].
   - XCHG: Обменивает данные между двумя операндами. Пример: XCHG EAX, EBX - меняет местами содержимое регистров EAX и EBX.
   - IN: Считывает данные из порта ввода/вывода в регистр AL, AX или EAX. Пример: IN AL, DX - считывает байт из порта с номером, указанным в DX, в регистр AL.
   - OUT: Записывает данные из регистра AL, AX или EAX в порт ввода/вывода. Пример: OUT DX, AL - записывает байт из регистра AL в порт с номером, указанным в DX
